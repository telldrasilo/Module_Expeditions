## 1. Введение и общая концепция

### 1.1 Назначение документа

Настоящий документ описывает архитектуру и содержание системы экспедиций (локаций, биомов, заданий и случайных событий) для idle-игры о кузнеце. Документ предназначен для разработчиков и содержит полное описание механик, структур данных, примеров и плана реализации.

### 1.2 Общая концепция игры

Игрок выступает в роли **кузнеца**, владеющего собственной мастерской в мире фэнтезийного средневековья. Кузнец не путешествует сам, а сотрудничает с **Гильдией Искателей** — организацией, выполняющей опасные поручения по добыче ресурсов, разведке и зачистке территорий.

**Ключевые особенности:**

- **Idle-механика:** игрок выбирает задание на доске, предоставляет оружие (и, возможно, расходники) и отправляет NPC-искателя на миссию. Прогресс идёт в реальном времени (таймер от 1 часа до нескольких часов).
- **Текстовое повествование:** в ходе миссии возникают случайные события, отображаемые в логе под таймлайном. События влияют на добычу, состояние искателя и оружия.
- **Стихийные шрамы:** столкновение оружия со стихиями (в бою или окружении) оставляет скрытые «шрамы». После возвращения кузнец может изучить повреждения и впоследствии пробудить «Душу Войны» оружия, превратив шрамы в полезные зачарования.
- **Полупроцедурная генерация контента:** задания и события собираются из предопределённых шаблонов и сущностей, привязанных к биомам, тирам и стихиям, что обеспечивает вариативность без потери логичности мира.

### 1.3 Основные модули системы

- **Биомы и локации:** 12 биомов из каталога `RESOURCE_DEPOT`, каждый с набором локаций, распределённых по 5 тирам сложности.
- **Стихии:** 13 стихий, влияющих на свойства оружия и окружения.
- **Задания:** генерируются на основе шаблонов с привязкой к биому, тиру, заказчику, врагам и ресурсам.
- **Случайные события:** происходят во время экспедиции, подразделяются на категории (враги, природные аномалии, руины, NPC, загадки) и имеют различные исходы.

## 2. Лор и сеттинг мира

### 2.1 Общая картина

Мир — **Королевство Аркедон** (рабочее название), средневековое феодальное государство, пережившее недавнюю войну за престол. Многие территории обезлюдели, границы баронств размыты, а дикие земли кишат чудовищами и аномалиями. Гильдия Искателей — полуофициальная организация, берущая на себя заказы на разведку, добычу редких материалов и уничтожение угроз.

Магия существует, но она не повсеместна и не «эпична». Стихии проявляются как естественные, хотя и опасные, силы: огненные жилы в вулканах, гнилостные испарения болот, искажения времени в древних руинах. Нет «избранных» или великих пророчеств — только люди, пытающиеся выжить и заработать.

### 2.2 Ключевые исторические пласты

#### 2.2.1 Древние цивилизации

- **Строители Монолитов** — давно исчезнувшая раса или культура, оставившая мегалитические сооружения в горах, лесах и пустынях. Их постройки отличаются идеальной геометрией и устойчивостью к стихиям. В руинах можно найти каменных стражей, артефакты с символикой магнетизма и кости.
- **Орден Светоносцев** — религиозная организация, процветавшая несколько столетий назад. Строили храмы-маяки на побережьях, аскетичные скиты в тундре и подземные святилища в кристальных пещерах. Их наследие связано со стихиями **Света** и **Тени** (как противопоставление).

#### 2.2.2 Недавняя история

- **Война Трёх Баронов** (закончилась около 20 лет назад) — междоусобный конфликт, опустошивший центральные регионы. Многие замки разрушены, а земли заросли лесом или превратились в болота. Остатки армий стали бандами мародёров, а некоторые командиры ушли в дикие земли.
- **Чума в Болотах** — эпидемия неизвестной болезни, поразившая восточные болота около 10 лет назад. Деревни вымерли, а местность стала ещё опаснее из-за расплодившихся гнилостных тварей.

### 2.3 География и ключевые регионы

Мир не имеет жёстко прописанной карты, но для целостности восприятия определены следующие регионы (провинции), объединяющие смежные биомы:

| Регион | Входящие биомы | Особенности |
|--------|----------------|-------------|
| **Восточные Марки** | `forest`, `swamp`, `bog` | Лесистая и болотистая местность, граничащая с горами. Много заброшенных деревень и сторожевых башен. |
| **Северные Пределы** | `tundra`, `mountains` (северные), `deep_mines` | Суровый край с залежами руд и древними шахтами. Редкое население, в основном звероловы и рудокопы. |
| **Пепельные Земли** | `volcanic`, `desert` (частично), `crystal_caves` | Геотермально активная область с вулканами, горячими источниками и кристальными гротами. |
| **Западное Побережье** | `ocean_coast`, `ancient_ruins`, `shadow_realm` (через порталы) | Скалистые берега, маяки Светоносцев, подводные пещеры и выходы в Теневой Мир. |
| **Центральные Баронства** | `forest`, `mountains` (предгорья), `desert` (южные) | Наиболее освоенная часть, где расположены города и мастерская игрока. |

### 2.4 Важные географические объекты

- **Река Серебрянка** — крупная водная артерия, текущая с гор через Восточные Марки и впадающая в океан на западе. На её берегах много торговых путей и руин.
- **Старый Тракт** — древняя дорога, соединяющая Центральные Баронства с Северными Пределами. Частично заброшена, но всё ещё используется караванами.
- **Пик Гроз** — высочайшая вершина в горах, постоянно окутанная бурями (связана со стихией **Молния**).
- **Чёрная Топь** — сердце болот, где по слухам находится логово древней твари, породившей чуму.

## 3. Биомы

### 3.1 Источник данных

Биомы определены в модуле `RESOURCE_DEPOT` как строковый юнион `Biome`. Для системы экспедиций используется тот же перечень для совместимости с каталогом материалов.

### 3.2 Полный список биомов

| ID (код) | Русское название (UI) | Описание для игрока |
|----------|----------------------|---------------------|
| `mountains` | Горы | Высокие скалистые пики, глубокие ущелья. Богаты рудами, но опасны обвалами и хищными птицами. |
| `forest` | Лес | Густые чащи, перемежающиеся полянами и ручьями. Много дичи, но и хищников. |
| `swamp` | Болото | Топкие трясины, заросшие камышом и мхом. Опасны ядовитыми испарениями и коварными тварями. |
| `volcanic` | Вулкан | Горячие пепельные пустоши, лавовые потоки. Источник редких огненных минералов. |
| `deep_mines` | Глубокие шахты | Заброшенные и действующие рудники, уходящие на огромную глубину. Темнота и спёртый воздух. |
| `desert` | Пустыня | Бескрайние пески, скальные выступы, оазисы. Днём — невыносимая жара, ночью — холод. |
| `tundra` | Тундра | Холодные равнины с вечной мерзлотой, редкими кустарниками и кочками. Обиталище выносливых зверей. |
| `ocean_coast` | Побережье | Скалистые утёсы, песчаные пляжи, прибрежные гроты. Богато морскими ресурсами, но подвержено штормам. |
| `ancient_ruins` | Древние руины | Остатки строений неизвестной цивилизации. Полны ловушек и магических аномалий. |
| `shadow_realm` | Теневой мир | Изломанное измерение, где тьма обретает форму. Искажённая физика и теневая флора/фауна. |
| `crystal_caves` | Кристальные пещеры | Подземные гроты, стены которых покрыты светящимися кристаллами. Хрупкая красота и опасная магия. |
| `bog` | Топь | Глубокие трясины с чёрной водой и гниющими деревьями. Отличается от `swamp` большей концентрацией Гнили и крови. |

| `deep_mines` | `mountains`, `crystal_caves`, `shadow_realm` (через глубины) |

| `shadow_realm` | `ancient_ruins`, `deep_mines`, `crystal_caves` |
| `crystal_caves` | `volcanic`, `deep_mines`, `shadow_realm` |

## 4. Система стихий

### 4.1 Роль стихий

Стихии — фундаментальные силы, проявляющиеся в окружении, врагах и ресурсах. Они не являются абстрактной магией, а скорее естественными аспектами мира, усиленными в определённых местах.

Для кузнеца стихии важны по двум причинам:

1. **Шрамы на оружии:** при столкновении со стихией оружие получает скрытое повреждение, которое после изучения и ремонта может быть преобразовано в зачарование.
2. **Тематические зачарования:** Душа Войны оружия, пробуждённая после накопления боевого опыта, даёт способности, основанные на комбинации шрамов.

### 4.2 Список стихий (13)

| № | Название (RU) | ID (EN) | Описание | Связанные биомы |
|---|--------------|---------|----------|-----------------|
| 1 | Пламя | `flame` | Живой огонь, пожирающий и очищающий. | `volcanic`, `desert`, `deep_mines` |
| 2 | Мороз | `frost` | Стужа, замедляющая время и крошащая сталь. | `tundra`, `deep_mines` (вечная мерзлота), `mountains` (вершины) |
| 3 | Молния | `lightning` | Чистая энергия, нестабильная и смертоносная. | `mountains` (грозы), `volcanic` (пепловые бури), `crystal_caves` |
| 4 | Гниль | `decay` | Распад, возвращение к земле, ядовитое плодородие. | `swamp`, `bog`, `forest` (глубины) |
| 5 | Кровь | `blood` | Жизненная сила, проклятия, связь с предками. | `ancient_ruins` (алтари), `shadow_realm`, `bog` |
| 6 | Кость | `bone` | Прочность, память формы, окаменелости. | `mountains`, `desert`, `tundra` |

| 10 | Искажение | `distortion` | Время, искривление реальности, эрозия. | `desert`, `shadow_realm`, `ancient_ruins` |
| 11 | Магнетизм | `magnetism` | Притяжение, отталкивание, скрытые связи. | `deep_mines`, `crystal_caves`, `mountains` |

### 4.3 Взаимодействие стихий (опционально)

Для будущей системы зачарований можно учесть парные отношения: например, `Пламя` и `Мороз` противоположны, `Кровь` и `Кость` усиливают друг друга. В рамках данного ТЗ эта механика не детализируется.

## 5. Локации и тиры сложности

### 5.1 Структура локации

Локация (Site) — конкретное место внутри биома, обладающее:

- Уникальным названием (например, «Сердце Корнегнилья»).
- Фиксированным тиром сложности (1–5).
- Набором характерных рисков (преобладающие стихии, типы врагов).
- Возможными условиями разблокировки (ключ, репутация, выполнение цепочки заданий).

### 5.2 Тиры сложности

| Тир | Описание | Доступность | Пример требований |
|-----|----------|-------------|-------------------|
| 1 | Безопасные окраины, простые поручения. | Открыт сразу для базовых биомов (`forest`, `mountains`, `swamp`). | Оружие любого качества. |
| 2 | Умеренно опасные зоны, требуется базовое снаряжение. | Открыт сразу для базовых биомов. | Оружие с хотя бы одним улучшением (или минимальный уровень искателя). |
| 3 | Опасные территории с агрессивной фауной и аномалиями. | Открывается после выполнения специального задания Гильдии (например, «Разведка: Восточные Марки»). | Оружие 2-го тира, возможно наличие ключа-карты. |
| 4 | Заброшенные крепости, логова редких монстров, глубокие руины. | Открывается после завершения нескольких заданий 3-го тира и получения соответствующего ключа/артефакта. | Оружие 3-го тира, специальный пропуск. |
| 5 | Легендарные места, доступные только подготовленным искателям. | Открывается через уникальные цепочки заданий, редкие ключи или глобальные события. | Оружие 4-го тира, уникальный ключ, высокая репутация. |

### 5.3 Ключи и условия разблокировки

Для открытия локаций высоких тиров или целых биомов используются следующие механики:

- **Карты и схемы:** предметы, выпадающие из сундуков или покупаемые у торговцев. Например, «Карта тайного прохода в Глубокие Шахты (тир 3)».
- **Артефакты-пропуски:** магические предметы, позволяющие пересечь барьер или договориться со стражем. Например, «Перо Хранителя Врат» для входа в `ancient_ruins` высокого тира.
- **Репутация и цепочки заданий:** выполнение серии поручений от конкретного заказчика открывает доступ к его личным угодьям или секретной информации.
- **Особые инструменты:** например, «Кирка с магнетитовым наконечником» для пробития заговорённого камня в `crystal_caves`.

Ключи являются **расходуемыми** или **постоянными** в зависимости от дизайна.

## 6. Доска заданий Гильдии Искателей

### 6.1 Механика взаимодействия

1. Игрок заходит в интерфейс **Гильдии Искателей** и видит **доску заданий**.
2. На доске отображается список доступных заданий (обычно 3–6 штук), которые обновляются раз в сутки (или по триггеру).
3. Каждое задание представлено **карточкой** с информацией.
4. Игрок выбирает задание и нажимает «Отправить искателя».
5. Система подбирает подходящего NPC-искателя (или игрок выбирает из доступных, если механика будет расширена).
6. Запускается таймер экспедиции.
7. В процессе появляются **случайные события** (текстовые отчёты).
8. По завершении игрок получает отчёт с добычей, состоянием оружия и наградой.

### 6.2 Структура карточки задания

| Элемент | Описание | Источник данных |
|---------|----------|-----------------|
| **Название** | Например: «Охота на огненную лису в Сердце Корнегнилья» | Генерируется по шаблону |
| **Биом и локация** | Название биома и конкретного места | Из справочника локаций |
| **Заказчик** | Имя и роль (например, «Торговец из Предместья») | Из пула NPC |
| **Описание** | Краткая сюжетная завязка (2-3 предложения) | Генерируется по шаблону |
| **Риски** | Иконки или названия стихий/врагов, представляющих опасность | Из локации и врагов |
| **Ожидаемые ресурсы** | Список материалов с примерной частотой (например, «Железная руда (часто), Огненная эссенция (редко)») | Фильтр по биому и тиру |
| **Требования** | Минимальный уровень оружия, наличие ключа | Из локации |
| **Базовая награда** | Золото, репутация Гильдии | Рассчитывается по формуле |
| **Длительность** | Базовое время экспедиции (например, 2 часа) | Зависит от тира и сложности |

### 6.3 Обновление доски

- Доска обновляется **ежедневно** в определённое время (например, в 00:00 по серверу).
- При обновлении старые невыполненные задания **исчезают** (если не предусмотрена система «сохранения» за плату).
- Генерация новых заданий происходит с учётом **открытых игроком биомов и тиров**, а также текущих **глобальных событий** (если реализованы).

## 7. Полупроцедурная генерация заданий

### 7.1 Общий принцип

Генерация заданий основана на **комбинировании предопределённых шаблонов** с данными из справочников (локации, враги, ресурсы, заказчики). Это гарантирует логическую непротиворечивость и высокую вариативность.

### 7.2 Шаблоны заданий

Каждый шаблон определяет **структуру** задания: тип миссии, формулу награды, требуемые данные.

**Примеры шаблонов:**

| ID шаблона | Тип | Шаблон названия | Шаблон описания | Базовая награда |
|------------|-----|-----------------|-----------------|-----------------|
| `hunt` | Охота | `Охота на {enemy} в {location}` | `{questGiver} просит уничтожить опасного зверя, терроризирующего окрестности {location}.` | Золото: 50 + тир*20; Репутация: 10 |
| `gather` | Сбор | `Сбор {resource} в {location}` | `{questGiver} нуждается в партии {resource} для своих нужд.` | Золото: 30 + редкость ресурса; Репутация: 5 |
| `explore` | Разведка | `Разведка: {location}` | `Гильдия хочет получить информацию о {location}. Необходимо провести разведку и отметить опасности.` | Золото: 80; Репутация: 15 + возможный ключ |
| `deliver` | Доставка | `Доставка груза через {location}` | `{questGiver} просит сопроводить ценный груз через опасный участок.` | Золото: 100; Репутация: 10 |
| `boss` | Уничтожение логова | `Зачистка: Логово {enemy} в {location}` | `Местные жители собрали средства на уничтожение логова {enemy}, обосновавшегося в {location}.` | Золото: 200; Репутация: 25; Уникальный ресурс |

### 7.3 Справочники для подстановки

#### 7.3.1 Заказчики (QuestGiver)

Статический пул NPC с именами, ролями и, возможно, предпочтениями по биомам.

```typescript
const questGivers = [
  { id: 'merchant_1', name: 'Торговец из Предместья', role: 'merchant' },
  { id: 'blacksmith_1', name: 'Старый кузнец Грегор', role: 'blacksmith' },
  { id: 'alchemist_1', name: 'Алхимик Лира', role: 'alchemist' },
  // ...
];
```

#### 7.3.2 Враги (Enemy)

Справочник врагов с привязкой к биомам, тирам и стихиям.

```typescript
interface Enemy {
  id: string;
  name: string;               // "Огненная лиса"
  description: string;        // "Хитрая тварь, чья шерсть тлеет в темноте."
  biomes: Biome[];            // ['forest', 'volcanic']
  tierRange: [number, number]; // [1, 3]
  elementalAffinity: Element[]; // ['flame']
  dangerLevel: 1..5;          // Влияет на сложность события
}
```

#### 7.3.3 Ресурсы (Resource)

Берутся из каталога `RESOURCE_DEPOT`. Для генерации заданий используется функция фильтрации по биому и тиру.

### 7.4 Алгоритм генерации задания

1. **Выбрать случайный шаблон** из списка доступных (с учётом весов, если нужно).
2. **Выбрать биом** из числа открытых игроком.
3. **Выбрать локацию** внутри биома с подходящим тиром (тир либо задаётся шаблоном, либо выбирается случайно из доступных).
4. **Подобрать врага/ресурс** в зависимости от типа шаблона:
   - Для `hunt`: фильтр врагов по биому и тиру локации.
   - Для `gather`: фильтр ресурсов по биому и тиру, выбор с учётом частоты (`frequency`).
5. **Выбрать заказчика** (можно привязать к биому или случайно).
6. **Сгенерировать тексты** путём подстановки значений в шаблоны.
7. **Рассчитать награду** по формулам шаблона с модификаторами тира и редкости.
8. **Сохранить задание** в состояние доски (до следующего обновления).

### 7.5 Кэширование и обновление

Сгенерированные задания хранятся в **состоянии игры** (например, в localStorage или на сервере) до момента обновления доски. При обновлении старые задания заменяются новыми.

## 8. Случайные события во время экспедиций

### 8.1 Общая механика

Во время активной экспедиции (пока идёт таймер) с определённой периодичностью (например, каждые 15–30 минут реального времени или каждый «тик» прогресса) генерируется **случайное событие**.

Событие выбирается из пула, отфильтрованного по:

- Биому и тиру текущей локации.
- Стихиям, присущим локации.
- Типу миссии (охота, сбор и т.д.) — опционально.
- Характеристикам искателя (в будущем).

Событие отображается в **логе** под таймлайном в виде короткого текста. Некоторые события имеют **игровые последствия**:

- Добавление ресурсов в добычу.
- Получение оружием **шрама** определённой стихии.
- Травма искателя (снижение эффективности, увеличение времени).
- Изменение длительности экспедиции.
- Нахождение ключа или карты.

### 8.2 Категории событий

#### 8.2.1 Враги (Combat)

Столкновение с враждебным существом. Может привести к бою (исход зависит от навыков искателя и оружия).

**Пример шаблона:**
> *«{seeker} наткнулся на {enemy}, сидящего в засаде. Завязалась короткая схватка.»*

**Возможные исходы:**
- Победа: получение ресурсов (шкура, клыки и т.п.), оружие получает шрам стихии врага.
- Тяжёлая победа: ресурсы, шрам, но искатель ранен (штраф к скорости).
- Поражение: искатель вынужден отступить, потеря части собранных ресурсов, оружие повреждено (доп. шрам).

#### 8.2.2 Природные аномалии (Environment)

Проявление стихийной силы окружения. Всегда оставляет **шрам** соответствующей стихии на оружии.

**Пример шаблона:**
> *«В расщелине скалы пульсирует жила чистого {element}. Воздух дрожит от жара.»*

**Исходы:**
- Искатель решает изучить: шрам `{element}` + возможный ресурс.
- Искатель обходит стороной: без последствий.

#### 8.2.3 Руины и следы прошлого (Remains)

Находка древних сооружений, тайников, артефактов.

**Пример:**
> *«{seeker} обнаружил старый алтарь с символами {element}. В углублении — горсть истлевшего пепла.»*

**Исходы:**
- Получение ресурса/ключа.
- Активация проклятия (шрам или штраф).
- Ничего не найдено.

#### 8.2.4 Нейтральные существа (Wildlife)

Встреча с неагрессивным, но потенциально полезным или опасным зверем.

**Пример:**
> *«На поляне пасётся {creature}. Его рога покрыты налётом сажи.»*

**Исходы:**
- Попытка подкрасться (шанс добыть редкий ресурс).
- Наблюдение (без последствий).

#### 8.2.5 Социальные встречи (NPC)

Короткое взаимодействие с другим человеком.

**Пример:**
> *«У костра сидит измождённый человек в прожжённом плаще. Он представляется наёмником из гильдии.»*

**Исходы:**
- Помочь: получить карту/информацию, репутацию.
- Проигнорировать.

#### 8.2.6 Загадки и испытания (Puzzle)

Ситуация, требующая проверки навыков искателя.

**Пример:**
> *«Вход в руины преграждает каменная дверь с углублениями в форме стихийных рун.»*

**Исходы (зависят от характеристик искателя):**
- Успех: доступ к тайнику с ценными ресурсами.
- Провал: ничего или ловушка (шрам, потеря ресурсов).

### 8.3 Структура данных события

```typescript
interface Encounter {
  id: string;
  category: 'combat' | 'environment' | 'remains' | 'wildlife' | 'npc' | 'puzzle';
  templateText: string;        // "{{seeker}} наткнулся на {{enemy}}..."
  allowedBiomes: Biome[];
  allowedTiers: LocationTier[];
  requiredElement?: Element;   // если событие связано с конкретной стихией
  possibleEnemies?: EnemyId[]; // для категории combat
  possibleResources?: ResourceId[]; // для remains, wildlife
  possibleOutcomes: EncounterOutcome[]; // варианты развития
}

interface EncounterOutcome {
  description: string;          // текст, добавляемый к основному шаблону
  chance: number;               // вероятность (0..1)
  effects: {
    addScar?: Element;          // добавить шрам оружию
    addResources?: { resourceId: string; amount: number }[];
    seekerInjury?: boolean;     // травма искателя
    timeModifier?: number;      // изменение оставшегося времени (в минутах)
    reputationChange?: number;
    keyItem?: string;           // ID ключа
  };
}
```

### 8.4 Связь со шрамами оружия

Шрам — это запись о том, что оружие подверглось воздействию определённой стихии. При ремонте после экспедиции кузнец может **изучить** шрам и либо устранить его, либо сохранить как скрытый потенциал.

**Механика (кратко, детали в другом модуле):**
- Каждое оружие имеет массив `scars: Element[]`.
- При событии, дающем шрам, в массив добавляется элемент стихии (если его ещё нет или счётчик увеличивается).
- После экспедиции кузнец видит список полученных шрамов и может выбрать действие.

## 10. Примеры реализации основных функций

### 10.1 Фильтрация материалов по биому и тиру

```typescript
import { getAllMaterials } from '@/modules/RESOURCE_DEPOT';

function getPossibleResourcesForLocation(
  biome: Biome,
  tier: LocationTier
): Array<{ material: Material; weight: number }> {
  const materials = getAllMaterials();
  const result: Array<{ material: Material; weight: number }> = [];

  for (const mat of materials) {
    const entry = mat.biomes.find(b => b.biome === biome);
    if (!entry) continue;

    // Базовый вес из частоты
    let weight = frequencyToWeight(entry.frequency);
    // Модификатор тира: на высоких тирах выше шанс редких ресурсов
    const tierModifier = 1 + (tier - 1) * 0.2; // 1.0, 1.2, 1.4, 1.6, 1.8
    weight *= tierModifier;

    result.push({ material: mat, weight });
  }

  return result;
}

function frequencyToWeight(freq: 'common' | 'uncommon' | 'rare' | 'legendary'): number {
  switch (freq) {
    case 'common': return 100;
    case 'uncommon': return 50;
    case 'rare': return 15;
    case 'legendary': return 3;
  }
}
```

### 10.2 Генерация задания

```typescript
function generateQuest(
  template: QuestTemplate,
  availableBiomes: Biome[],
  locations: ExpeditionLocation[],
  enemies: Enemy[],
  questGivers: QuestGiver[]
): GeneratedQuest | null {
  // 1. Выбрать случайный биом
  const biome = randomElement(availableBiomes);
  if (!biome) return null;

  // 2. Выбрать локацию нужного тира
  const tier = selectTierForQuest(template); // зависит от прогресса игрока
  const possibleLocations = locations.filter(loc => loc.biome === biome && loc.tier === tier);
  if (possibleLocations.length === 0) return null;
  const location = randomElement(possibleLocations);

  // 3. Выбрать заказчика
  const giver = randomElement(questGivers);

  // 4. Подставить данные в зависимости от типа
  let targetEnemyId: string | undefined;
  let targetResourceId: string | undefined;

  if (template.requiredDataType === 'enemy') {
    const possibleEnemies = enemies.filter(e => 
      e.biomes.includes(biome) && e.tierRange[0] <= tier && e.tierRange[1] >= tier
    );
    if (possibleEnemies.length === 0) return null;
    targetEnemyId = randomElement(possibleEnemies).id;
  } else if (template.requiredDataType === 'resource') {
    const resources = getPossibleResourcesForLocation(biome, tier);
    if (resources.length === 0) return null;
    targetResourceId = weightedRandom(resources).material.id;
  }

  // 5. Сгенерировать тексты
  const enemy = targetEnemyId ? enemies.find(e => e.id === targetEnemyId)! : null;
  const resource = targetResourceId ? getMaterialById(targetResourceId) : null;

  const title = fillTemplate(template.titleTemplate, {
    enemy: enemy?.name ?? '',
    resource: resource?.name ?? '',
    location: location.name,
  });

  const description = fillTemplate(template.descriptionTemplate, {
    questGiver: giver.name,
    enemy: enemy?.name ?? '',
    resource: resource?.name ?? '',
    location: location.name,
  });

  // 6. Рассчитать награду и длительность
  const reward = calculateReward(template, tier, enemy?.dangerLevel ?? 1);
  const duration = calculateDuration(tier, template.type);

  // 7. Собрать подсказки по ресурсам
  const resourceHints = generateResourceHints(location, tier);

  return {
    id: generateQuestId(),
    templateId: template.id,
    locationId: location.id,
    questGiverId: giver.id,
    title,
    description,
    targetEnemyId,
    targetResourceId,
    reward,
    duration,
    resourceHints,
  };
}
```

```typescript
): EncounterResult {
  // Фильтруем доступные события
  const possible = allEncounters.filter(enc => 
    enc.allowedBiomes.includes(location.biome) &&
    enc.allowedTiers.includes(location.tier) &&
    (!enc.requiredElement || location.risks.includes(enc.requiredElement))
  );

  if (possible.length === 0) {
    return { text: 'Всё идёт спокойно...', effects: {} };
  }

  const encounter = weightedRandom(possible, e => e.weight ?? 1);
  
  // Выбираем исход
  const outcome = weightedRandom(encounter.outcomes, o => o.chance);
  
  // Заполняем шаблон
  const context = buildContext(encounter, location, seeker);
  let fullText = fillTemplate(encounter.templateText, context);
  fullText += ' ' + outcome.description;

  return {
    text: fullText,
    effects: outcome.effects,
  };
}
```
